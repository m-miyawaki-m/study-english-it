# Code Review

未コミットの変更に対するセキュリティ・品質レビューを実施する。

## 手順

1. 変更ファイルを取得: `git diff --name-only HEAD`

2. 各ファイルをチェック:

**セキュリティ (CRITICAL):**
- ハードコードされたAPIキー・パスワード・トークン
- SQLインジェクション脆弱性
- XSS脆弱性
- 入力バリデーション不足
- パストラバーサルリスク

**コード品質 (HIGH):**
- 関数が50行超
- ファイルが800行超
- ネストが4段超
- エラーハンドリング不足
- console.log / System.out.println の残留
- TODO/FIXME コメント

**ベストプラクティス (MEDIUM):**
- ミュータブルなパターン（イミュータブルに変更すべき）
- 新コードにテストがない
- 不要なコメントやデッドコード

3. レポートを生成:
   - 重大度: CRITICAL, HIGH, MEDIUM, LOW
   - ファイル位置と行番号: `file.java:123`
   - 問題の説明
   - 修正案

4. CRITICAL または HIGH の問題がある場合はコミットを止める

セキュリティ脆弱性のあるコードは絶対に承認しない。
